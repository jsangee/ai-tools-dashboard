💻 VS Code (Visual Studio Code)

A free, powerful text editor that's become the industry standard for writing and managing code.

What it does

VS Code is a sophisticated text editor built by Microsoft specifically for writing code. But it's far more than just a notepad—it understands different programming languages, highlights syntax to make code readable, catches errors before you run the code, suggests completions as you type, and integrates with thousands of tools through extensions. It can connect to databases, cloud services, version control systems, and testing frameworks. Think of it as the central workspace where developers spend most of their day.

Why it matters for me

Understanding VS Code helps you grasp how modern development works. When your team talks about "extensions," "debugging," or "IntelliSense," they're referring to VS Code features. It's free and cross-platform (Windows, Mac, Linux), which means your entire team can use the same tools regardless of their hardware. It's also highly customizable, so your developers can optimize it for your specific tech stack. When evaluating project timelines or tooling costs, knowing that VS Code is the foundation helps you understand what's realistic and what additional tools might be needed.

Analogy: VS Code is like a high-end architect's drafting table with built-in measuring tools, automatic error-checking, and the ability to connect to every supplier and contractor you might need—all in one workspace. It's where the blueprints (code) get created and refined.

☁️ Salesforce CLI (sf / sfdx)

A command-line tool that lets developers interact with Salesforce orgs directly from their terminal—no clicking through the web interface.

What it does

The Salesforce CLI is a specialized command-line tool built by Salesforce for developers working on the Salesforce platform. It allows you to create scratch orgs (temporary Salesforce environments for testing), deploy code to orgs, retrieve metadata, run tests, query data, and automate repetitive tasks—all from the terminal. Instead of logging into Salesforce and clicking through Setup, developers can type commands like sf org create scratch or sf project deploy start to get work done faster. It integrates tightly with developer workflows and CI/CD pipelines. The commands start with either sf (newer version) or sfdx (legacy version).

Why it matters for me

The Salesforce CLI represents the shift from manual, click-based Salesforce administration to automated, code-based development. When your team talks about "DevOps," "CI/CD," or "source control" for Salesforce projects, the CLI is often the foundation. It enables developers to version control their Salesforce configurations, automate deployments between environments (dev → test → production), and work locally on their machines before pushing changes to the cloud. For you, understanding the CLI means you can appreciate the difference between traditional admin work (clicking in Setup) and modern development practices (scripted, repeatable, testable). It also helps you understand project velocity—teams using the CLI can move much faster than those relying solely on manual processes.

Analogy: If Salesforce Setup is like manually configuring every light switch and thermostat in a building by walking around with a clipboard, the Salesforce CLI is like having a central control panel where you can script all those settings, save them, and instantly replicate them across dozens of buildings—fast, consistent, and reversible.

✨ Cursor

An AI-powered code editor built on VS Code that lets you code by having conversations with AI.

What it does

Cursor takes everything great about VS Code and adds deeply integrated AI capabilities. Instead of just writing code manually, you can describe what you want in plain English, and Cursor's AI will generate, modify, or explain the code for you. It can understand your entire codebase, suggest changes across multiple files, catch bugs before they happen, and even refactor complex code. The AI assistant sees what you're working on and can proactively suggest improvements. It's like having an expert pair programmer sitting next to you at all times.

Why it matters for me

Cursor represents a fundamental shift in how software gets built. For non-technical leaders or those learning to code, it dramatically lowers the barrier to entry—you can build functional applications by describing what you want rather than memorizing syntax. For experienced developers, it multiplies productivity by handling boilerplate code, catching edge cases, and speeding up debugging. When evaluating team composition or project feasibility, knowing that AI-assisted tools like Cursor exist means smaller teams can accomplish more, and non-developers can contribute more directly to technical projects. This is the future of development—AI and humans collaborating.

Analogy: If VS Code is the architect's drafting table, Cursor is that same table but with an expert architect standing next to you, watching what you draw, suggesting better approaches, fixing mistakes in real-time, and sometimes just taking the pencil and drawing exactly what you described verbally.

🤖 Claude

An AI assistant that understands context, writes code, analyzes data, and helps solve complex problems through conversation.

What it does

Claude is an AI assistant developed by Anthropic that you can have natural conversations with. Think of it as a highly knowledgeable colleague who can write documents, analyze information, write and debug code, explain complex concepts, and help you think through problems. Unlike simple chatbots, Claude understands nuance, maintains context throughout long conversations, and can handle sophisticated tasks like reviewing contracts, writing technical documentation, or architecting software solutions.

Why it matters for me

As a director, Claude can dramatically accelerate your team's productivity. It can help draft RFP responses, review technical proposals, explain technical concepts to non-technical stakeholders, analyze project risks, generate project documentation, and even help with code reviews. It's particularly valuable because it can work across different domains—business strategy, technical implementation, and communication—making it a versatile tool for leadership. Many organizations are integrating AI assistants like Claude into their workflows to reduce repetitive work and free up humans for higher-value decision-making.

Analogy: Claude is like having a brilliant consultant who never sleeps, never forgets what you discussed, and can instantly switch between being a lawyer, engineer, writer, or strategist depending on what you need—all in the same conversation.

🏢 How Salesforce Wants You to Use Claude

Salesforce is making Claude available to everyone—not just engineers—but through your IDE, not as a standalone app.

What it does

Salesforce leadership (including Srini Tallapragada) has announced that Claude is now available company-wide, not just for engineering teams. However, the strategic direction is to use Claude inside your development environment (VS Code, Cursor, or other IDEs) rather than the standalone desktop app or browser version at claude.ai. This approach keeps Claude connected to your workspace, giving it full context of the code and files you're working with. The IDE integration means Claude can read your project files, suggest edits directly in your code, understand your Salesforce org metadata, and help you build solutions while staying inside your primary work environment.

Why it matters for me

This isn't just a tooling preference—it's a strategic shift in how Salesforce expects all employees (technical and non-technical) to work. By embedding Claude in the IDE, Salesforce is encouraging everyone to think more like developers: working with code in context, building solutions iteratively, and leveraging AI that understands your entire project—not just isolated questions. For directors and business leaders, this means:

  • Context-aware assistance: Claude sees your Salesforce metadata, your project documents, your org configuration—so its suggestions are specific to your actual work, not generic.
  • Democratized development: Non-engineers can now build functional solutions (flows, automation, reports) by describing what they need in plain English while Claude handles the technical syntax.
  • Unified workflow: Instead of switching between browser tabs, Slack, Claude.ai, and your IDE, everything happens in one place—making you faster and reducing context switching.
  • Enterprise governance: When Claude runs inside your IDE connected to Salesforce systems, the company can better manage security, compliance, and data access compared to standalone web apps.

The message from leadership is clear: this is how modern work gets done at Salesforce. Learning to work in an IDE with Claude isn't optional—it's becoming the standard way to operate, regardless of your role. The barrier to entry is lower than you think, and the productivity gains are substantial once you make the shift.

Analogy: Think of it this way: the browser version of Claude is like calling a consultant on the phone and describing your problem from memory. Claude in your IDE is like inviting that consultant into your office, showing them all your files, spreadsheets, and systems, and having them work alongside you with full visibility—they can see what you see, edit directly in your documents, and give advice that's grounded in your actual situation, not abstract scenarios.

📚 Additional Context: Foundational Concepts

📁 Workspace

What it is: An organized folder containing all the files, code, and resources for a specific project—your project's home base.

Why it matters: When you open a workspace in VS Code or Cursor, the editor understands the entire project structure. This is why AI tools like Claude can give context-aware suggestions—they see the whole picture, not just one file. Workspaces are how developers organize work and maintain boundaries between different projects.

Analogy: A workspace is like a war room for a specific initiative—all blueprints, contracts, and resources for that project organized in one place.

⌨️ Terminal / CLI

What it is: A text-based interface for giving direct commands to your computer—no mouse, no graphics, just typed instructions.

Why it matters: The terminal is how developers run commands efficiently and automate repetitive tasks. When your team mentions "CI/CD pipelines," "deployment scripts," or "running tests," they're often describing terminal commands running automatically. Common commands include cd (change directory), ls (list files), and git (version control).

Analogy: The terminal is like texting your computer instead of tapping through apps—direct communication with no middleman interface.
